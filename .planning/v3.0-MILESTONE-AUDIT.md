## Tech Debt Summary

No critical blockers. The following non-critical items are deferred:

### Phase 6 — Cookie-based Auth & CSRF Protection
- **⚠ `@app.on_event("startup")` deprecated** — Should migrate to FastAPI `lifespan` context manager in a future cleanup phase.
- **⚠ `UserResponse` class-based Pydantic config** — Deprecated since Pydantic v2; should migrate to `model_config = ConfigDict(...)` pattern.

### Phase 7 — Secure Headers & Rate Limiting
- **ℹ CSP header** — Current `default-src 'self'` will need updating if external CDN fonts/icons are added.
- **ℹ In-memory rate limiter** — Sufficient for single-instance local tool; Redis backend needed for horizontal scaling in production.

### Phase 8 — Input Hardening & Role Authorization
- **ℹ No admin UI for role promotion** — Roles can only be set via direct DB access or registration payload. An admin panel for role management is out of scope for v3 but should be planned for v4.

**Total tech debt items: 5 across 3 phases** — No blockers; all deferred by design.
