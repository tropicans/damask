<threat_model>
### Security Threat Model (ASVS L1)
- **Threat**: PII leakage through persistent mapping files.
  - **Mitigation**: All mappings are processed in-memory (RAM) only and never written to the server's local storage/disk.
- **Threat**: Collision in generated fake values violating bijectivity.
  - **Mitigation**: Implement a retry loop (up to 100 times) with a unique counter suffix fallback to ensure uniqueness.
- **Threat**: Mismatched or corrupted values during reversion leading to incorrect or partially restored data.
  - **Mitigation**: Fail-fast validation checking that every cell value exists in the mapping key, aborting the process if not.
</threat_model>
