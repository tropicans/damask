<threat_model>
### Security Threat Model (ASVS L1)
- **Threat**: Lost reversion key leading to unrecoverable data.
  - **Mitigation**: Success screen includes prominent warning details notifying users that the server stores nothing and the key must be kept locally in a safe location.
</threat_model>

<task type="auto">
  <name>Extend maskFile API function in frontend client</name>
  <files>frontend/src/api/mask.ts</files>
  <action>
    Modify `maskFile` function in `frontend/src/api/mask.ts`:
      - Add parameter `generateKey: boolean = false`.
      - Append `generate_key` (as string `"true"` or `"false"`) to the `formData` object.
      - Return the backend response which will be a ZIP archive blob if `generateKey` was checked, or a standard CSV/XLSX blob if not.
  </action>
  <verify>
    Check that typescript compiles without errors.
  </verify>
  <acceptance_criteria>
    - `maskFile` accepts `generateKey` parameter.
    - `formData` correctly appends `generate_key` parameter.
  </acceptance_criteria>
  <done>Client API helper for file masking is extended.</done>
</task>

<task type="auto">
  <name>Add key generation checkbox and success screen to App.tsx</name>
  <files>frontend/src/App.tsx</files>
  <action>
    Modify `frontend/src/App.tsx`:
      - Add a state variable `generateKey: boolean` (default false) and `successData: { blob: Blob; filename: string; generatedKey: boolean } | null` (default null).
      - In the masking configuration panel (near the start masking button), add an opt-in checkbox labeled "Buat Kunci Pemulihan" / "Generate Reversion Key" styled with Tailwind, updating the `generateKey` state.
      - Update `handleMaskExecute`:
        - Pass the `generateKey` state to `maskFile(file, selectedRules, generateKey)`.
        - Upon success, instead of dynamically clicking an anchor to download the file directly, save the response data (the blob, filename, and generateKey indicator) in the `successData` state.
      - Add conditional rendering: if `successData` is not null, display a dedicated "Success" panel instead of the normal masking view.
      - The success screen must display:
        - Big success checkmark icon.
        - Heading: "Penyamaran Data Berhasil" (Masking Successful).
        - Description details about the processed file.
        - If `generateKey` was checked (ZIP file returned):
          - A button to "Download Paket (ZIP)" (downloading the ZIP file containing both files).
          - A security alert card in red/amber border highlighting: "Kunci pemulihan ini bersifat lokal dan tidak disimpan di server. Pastikan Anda menyimpannya dengan aman. Jika hilang, data asli tidak dapat dipulihkan."
        - If `generateKey` was not checked:
          - A button to download the masked file.
        - A button to "Kembali ke Masking" to reset the states (`handleClear` + clear `successData`) and return to the engine screen.
  </action>
  <verify>
    Verify that App compiles and the state transitions correctly from configuration to success.
  </verify>
  <acceptance_criteria>
    - Checkbox is displayed and functional in the configuration panel.
    - Masking transitions to the success screen with download triggers.
    - Security warnings are visible if a reversion key was requested.
  </acceptance_criteria>
  <done>App.tsx is updated with masking key configuration and success view transitions.</done>
</task>
