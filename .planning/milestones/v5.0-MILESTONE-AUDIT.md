## Integration & E2E Flows Checked

1. **Authentication Flow:** Registration blocks without invite token -> Registration fails on weak passwords -> Registration succeeds with valid invite -> Lockout kicks in after 5 incorrect logins.
2. **Admin Controls Flow:** Deactivation logs out user immediately -> Deactivated user blocked from logging back in -> Admin role promote/demote updates permissions.
3. **Database Integration Flow:** Retry helper handles database initialization lags in Postgres -> Pytest suite uses centralized session for consistent transactions.
4. **Proxy & Security Headers Flow:** Nginx proxies api requests -> static asset caching headers are set -> file upload sizes restricted to 50M.

All flows verified as fully operational and secure.
