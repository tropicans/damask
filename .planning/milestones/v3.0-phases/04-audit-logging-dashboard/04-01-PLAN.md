    Create a new file `backend/app/models/job.py` with SQLModel table classes `MaskingJob` (representing table 'masking_jobs') and `JobDetail` (representing table 'job_details').
    `MaskingJob` fields:
    - `id`: str (primary key, uuid)
    - `user_id`: str (foreign_key="users.id" with ondelete="CASCADE", indexed)
    - `file_name`: str
    - `file_size_bytes`: int
    - `row_count`: Optional[int]
    - `status`: str (either "SUCCESS" or "FAILED")
    - `error_message`: Optional[str]
    - `created_at`: datetime (default_factory=datetime.utcnow)
    
    `JobDetail` fields:
    - `id`: str (primary key, uuid)
    - `job_id`: str (foreign_key="masking_jobs.id" with ondelete="CASCADE", indexed)
    - `column_name`: str
    - `rule_name`: str (directly as string)

    Register these models in `backend/app/db.py` by importing them in `init_db` (using import `from app.models.job import MaskingJob, JobDetail`).
    Ensure SQLite foreign key support is enabled on engine connection. In `backend/app/db.py`, listen for connect event on Engine to execute "PRAGMA foreign_keys=ON".
  </action>
  <verify>Run mypy or compile check to ensure SQLModel tables are valid.</verify>
  <acceptance_criteria>

    - backend/app/db.py imports MaskingJob and JobDetail inside init_db
    - backend/app/db.py contains pragma listener for foreign_keys=ON
  </acceptance_criteria>

  <action>
    Create a new file `backend/app/api/endpoints/jobs.py` exposing:
    - `GET /` (lists user masking jobs): queries `MaskingJob` filtered by `user_id = current_user.id`, ordered by `created_at` descending, with pagination parameters `skip: int = 0` and `limit: int = 10`.
    - `GET /stats` (aggregates metrics): queries total files (count of SUCCESS jobs), total rows sanitized (sum of row_count for SUCCESS jobs), and success rate (count of SUCCESS jobs / count of all jobs).
    - `GET /{job_id}/details` (lists columns masked for a job): queries `JobDetail` for a given `job_id`, asserting that the job belongs to `current_user.id` to prevent access to other users' data.
    
    All endpoints must require authentication via `Depends(get_current_user)`.
    Include the new router in `backend/app/api/api.py` under prefix `/jobs`.
  </action>
