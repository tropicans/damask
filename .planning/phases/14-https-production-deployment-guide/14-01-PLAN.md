<threat_model>
### Security Threat Model (ASVS L1)
- **Threat**: Direct external traffic bypassing Nginx to access frontend or backend containers directly.
  - **Mitigation**: Bind Nginx port 80 to localhost (127.0.0.1:80:80) and do not expose frontend (5173) or backend (8000) ports to the host or internet.
- **Threat**: File upload vulnerability (Denial of Service via extremely large uploads).
  - **Mitigation**: Set Nginx client_max_body_size to 50M to enforce the project constraint (max 50MB file size).
</threat_model>

  <name>Create Nginx Configuration Template</name>
  <files>nginx/nginx.conf</files>
  <read_first>nginx/nginx.conf</read_first>
  <action>
    Create a new directory nginx if it does not exist, and write the Nginx configuration template to nginx/nginx.conf.
    The configuration must:
    - Set client_max_body_size to 50M to handle up to 50MB file uploads.
    - Set up upstream blocks for backend (backend:8000) and frontend (frontend:5173).
    - Route /api requests to the backend upstream.
    - Route all other requests to the frontend upstream.
    - Cache frontend static assets (/assets/) with a long expiration (1y) and "public, no-transform" Cache-Control header.
    - Ensure index.html and other paths are not cached ("no-cache, no-store, must-revalidate").
    - Include standard reverse proxy headers (X-Real-IP, X-Forwarded-For, X-Forwarded-Proto, Host).
  </action>
  <verify>
    Verify that nginx/nginx.conf is written correctly.
  </verify>
  <acceptance_criteria>
    - nginx/nginx.conf contains client_max_body_size 50M.
    - nginx/nginx.conf routes /api to http://backend_server.
    - nginx/nginx.conf routes / to http://frontend_server.
    - nginx/nginx.conf sets Cache-Control for assets and no-cache for other routes.
  </acceptance_criteria>
  <done>Nginx configuration template created.</done>
</task>
