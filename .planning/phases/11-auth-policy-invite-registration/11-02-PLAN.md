<threat_model>
### Security Threat Model (ASVS L1)
- **Threat**: Invite token exposed in browser history via URL parameter.
  - **Mitigation**: This is acceptable for an internal tool. After successful registration, the token is consumed server-side. Frontend does not store the token in localStorage.
- **Threat**: Admin invite link shared unintentionally via copy-paste.
  - **Mitigation**: Tokens expire in 48 hours and are single-use. A security reminder is shown in the invite modal.
</threat_model>

    2. **Password strength calculation utility** (add before component):
    ```typescript
    interface PasswordStrength {
      level: 'empty' | 'lemah' | 'sedang' | 'kuat';
      score: number; // 0-4 criteria met
      label: string;
    }

    const calculatePasswordStrength = (password: string): PasswordStrength => {
      if (!password) return { level: 'empty', score: 0, label: '' };
      let score = 0;
      if (password.length >= 8) score++;
      if (/[A-Z]/.test(password)) score++;
      if (/[a-z]/.test(password)) score++;
      if (/[0-9]/.test(password)) score++;
      if (score <= 1) return { level: 'lemah', score, label: 'Lemah' };
      if (score <= 3) return { level: 'sedang', score, label: 'Sedang' };
      return { level: 'kuat', score, label: 'Kuat' };
    };
    ```
