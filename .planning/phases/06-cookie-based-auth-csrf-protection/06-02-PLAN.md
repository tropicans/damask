<tasks>
<task type="auto">
  <name>Implement backend CSRF validation middleware</name>
  <files>backend/app/main.py</files>
  <action>
    Modify `backend/app/main.py`.
    Create a custom ASGI middleware or standard Starlette `BaseHTTPMiddleware` called `CSRFMiddleware`.
    In the middleware:
      - For safe methods (GET, HEAD, OPTIONS):
        - Process request, and if the client does not already have a `secure_data_csrf` cookie, generate a secure random token (e.g. `secrets.token_urlsafe(32)`) and set it via `response.set_cookie(key="secure_data_csrf", value=token, httponly=False, secure=settings.COOKIE_SECURE, samesite="lax")` (note httponly=False is required for JS to read it).
      - For mutating methods (POST, PUT, DELETE):
        - Check if request path is `/api/auth/login` or `/api/auth/register`. If so, bypass validation and process the request.
        - Otherwise, extract `secure_data_csrf` from `request.cookies` and `X-CSRF-Token` from `request.headers`.
        - Compare them. If either is missing, or they do not match, return a `JSONResponse(status_code=403, content={"detail": "CSRF token validation failed."})`.
    Register the `CSRFMiddleware` on the FastAPI `app` instance.
  </action>
  <verify>
    Ensure backend starts and compiles successfully.
  </verify>
  <acceptance_criteria>
    - CSRFMiddleware is registered in main.py.
    - CSRFMiddleware sets secure_data_csrf cookie on GET requests (httponly=False).
    - CSRFMiddleware rejects mutating requests with mismatched/missing tokens with 403 Forbidden.
  </acceptance_criteria>
  <done>Backend CSRF middleware is implemented and active.</done>
</task>

  </action>
