<threat_model>
### Security Threat Model (ASVS L1)
- **Threat**: Disk leak of ZIP/mapping files on the server.
  - **Mitigation**: Perform all ZIP compression and JSON serialization strictly in memory using `zipfile` and `io.BytesIO`.
- **Threat**: Directory traversal (Zip Slip) vulnerability when unzipping or naming ZIP entries.
  - **Mitigation**: Clean and sanitize all entry names written into the ZIP file, using simple relative basenames only.
</threat_model>

<task type="auto">
  <name>Extend the /mask endpoint to accept generate_key Form parameter</name>
  <files>backend/app/api/endpoints/mask.py</files>
  <action>
    Modify the `mask_file` endpoint in `backend/app/api/endpoints/mask.py`:
      - Add `generate_key: bool = Form(False)` to the endpoint parameters.
      - If `generate_key` is True, invoke `mask_dataframe(df, rules_dict, return_mappings=True)` to get `masked_df` and the bijective `mappings`.
      - Generate the masked file bytes in memory (CSV or Excel).
      - If `generate_key` is True:
        - Create a ZIP archive in-memory using `io.BytesIO` and `zipfile.ZipFile`.
        - Write the masked file to the ZIP archive under filename `{base_name}_masked{ext}`.
        - Write the stringified mappings dictionary to the ZIP archive under filename `{base_name}_reversion_key.json` using `json.dumps`.
        - Set response headers for filename `{base_name}_masked.zip` and media type `"application/zip"`.
        - Stream the ZIP archive back.
      - Otherwise, continue with the standard single-file StreamingResponse.
  </action>
