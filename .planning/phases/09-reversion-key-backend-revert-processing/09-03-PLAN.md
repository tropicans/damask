<threat_model>
### Security Threat Model (ASVS L1)
- **Threat**: Denial of Service (DoS) via massive file upload on revert endpoint.
  - **Mitigation**: Implement chunked file size validation rejecting uploads exceeding 50MB before parsing.
- **Threat**: Malformed JSON key causing server crash or unhandled exception.
  - **Mitigation**: Wrap JSON parsing in try-except block, returning HTTP 400 Bad Request with a clear user-friendly error message.
- **Threat**: CSRF / Rate limiting bypass on revert endpoint.
  - **Mitigation**: Add `@limiter.limit("10/minute")` and require authenticated session cookies via `Depends(get_current_user)`.
</threat_model>

<task type="auto">
  <name>Implement the /mask/revert endpoint</name>
  <files>backend/app/api/endpoints/mask.py</files>
  <action>
    Add the `revert_file` endpoint `/revert` in `backend/app/api/endpoints/mask.py`:
      - Path: `@router.post("/revert")`
      - Security: Apply `@limiter.limit("10/minute")` and `Depends(get_current_user)`.
      - Parameters:
        - `request: Request`
        - `file: UploadFile = File(...)`
        - `key: UploadFile = File(...)`
        - `session: Session = Depends(get_session)`
      - Validations:
        - Verify file extension is `.csv` or `.xlsx`.
        - Validate file size is <= 50MB using the chunked reading loop.
        - Validate key file size is <= 10MB (to prevent massive JSON payloads).
      - Parsing:
        - Read and parse key file bytes as JSON to obtain the `mappings` dictionary. Raise HTTP 400 if JSON parsing fails.
        - Parse the file into a DataFrame `df` (using `pd.read_csv` or `pd.read_excel` with `dtype=str`).
      - Processing:
        - Call `revert_dataframe(df, mappings)` inside a try-except block.
        - Catch `ValueError` raised by `revert_dataframe` and raise HTTP 400 Bad Request with the detailed error message.
      - Streaming:
        - Write the restored DataFrame to a memory buffer (`io.BytesIO`) in its original format.
        - Return the buffer as a StreamingResponse with appropriate media type and content-disposition header (`{base_name}_reverted{ext}`).
  </action>
  <verify>
    Verify that backend compiles and routing is active.
  </verify>
  <acceptance_criteria>
    - `/revert` endpoint exists and requires authentication.
    - Successfully reverts files and streams them back.
    - Returns HTTP 400 Bad Request with descriptive errors on mismatch or validation failure.
  </acceptance_criteria>
