Product Requirement Document (PRD)Nama Proyek: LLM Data Masker Web Tool (SecureData Web)Status: Draft / ProposalTahun: 20261. Ringkasan Eksekutif & Latar BelakangBanyak pengguna ingin memanfaatkan LLM untuk menganalisis data bisnis atau riset mereka. Namun, mengunggah berkas CSV/XLSX yang mengandung Data Pribadi (PII) atau informasi finansial sensitif melanggar kebijakan kepatuhan data (seperti GDPR atau UU PDP). SecureData Web adalah alat berbasis web internal/lokal yang memungkinkan pengguna mengunggah berkas, melakukan pemetaan (mapping) kolom yang ingin disamarkan, dan mengunduh berkas yang telah aman (masked) secara instan tanpa menyimpan data asli pengguna di server.2. Tujuan & Sasaran (Objectives)Keamanan Data: Memastikan data sensitif tidak pernah keluar dari lingkungan aman sebelum diunggah ke LLM.Kemudahan Penggunaan: Menyediakan antarmuka drag-and-drop berkas untuk pengguna non-programmer.Fleksibilitas: Mengizinkan pengguna memilih jenis masking per kolom (Nama, Email, Telepon, Rentang Angka, atau Dilewati/Abaikan).Efisiensi: Proses masking dilakukan secara cepat dengan streaming atau pemrosesan memori sementara (RAM) tanpa penyimpanan permanen berkas mentah.3. Alur Pengguna (User Flow)Pengguna membuka halaman utama web.Pengguna mengunggah berkas CSV atau XLSX.Sistem membaca struktur tabel dan menampilkan daftar kolom beserta sampel data asli (baris pertama saja sebagai pratinjau).Sistem memberikan rekomendasi jenis masking secara otomatis (berdasarkan nama kolom).Pengguna dapat mengubah atau menyesuaikan jenis masking per kolom melalui menu dropdown.Pengguna mengklik tombol "Mulai Masking".Sistem memproses data dan langsung memicu unduhan (download) berkas baru yang telah disamarkan.4. Persyaratan Fungsional (Functional Requirements)FR-1: Manajemen Berkas (File Management)Sistem harus menerima unggahan format .csv, .xlsx, dan .xls.Ukuran maksimal berkas dibatasi (misal: 50MB) untuk mencegah kelebihan beban memori server.Berkas yang diunggah harus diproses di dalam memory buffer (RAM) dan tidak boleh disimpan secara permanen di penyimpanan lokal server (demi keamanan).FR-2: Deteksi & Pratinjau Kolom (Column Detection & Preview)Sistem harus membaca baris pertama berkas untuk mengambil nama-nama kolom.Sistem harus mengambil maksimal 3 baris pertama data sebagai contoh visual (pratinjau) bagi pengguna.Sistem harus menerapkan logika regex (seperti pada skrip Python sebelumnya) untuk memberikan rekomendasi tipe masking otomatis (misal: kolom "Email_Customer" otomatis diset ke opsi "Mask As Email").FR-3: Konfigurasi Masking oleh PenggunaPengguna dapat memilih tipe masking untuk setiap kolom melalui antarmuka web dengan opsi:No Masking (Biarkan data asli)Fake Name (Nama acak Indonesia/Global)Fake Email (Email acak)Fake Phone (Nomor telepon acak)Anonymize ID/Number (Mengacak digit angka/karakter)Perturb Numeric (Mengacak angka keuangan dengan variasi $\pm 20\%$)FR-4: Riwayat Aktivitas & Dasbor (Opsional - Jika menggunakan Login)Jika sistem diimplementasikan dengan akun pengguna, sistem harus mencatat riwayat aktivitas (Nama file, tanggal pemrosesan, jumlah baris yang dimasking). Catatan: Isi data berkas tidak dicatat, hanya metadatanya saja.5. Persyaratan Non-Fungsional (Non-Functional Requirements)Keamanan: Enkripsi HTTPS wajib digunakan. Semua temporary file di memori harus dihapus langsung setelah proses download selesai atau session berakhir.Teknologi Rekomendasi: Backend menggunakan Python (FastAPI atau Flask) karena integrasi pandas dan faker sangat efisien di Python. Frontend menggunakan React.js atau Streamlit (untuk pengembangan cepat).
